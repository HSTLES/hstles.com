Architecture Overview

Two layers working together. Smart separation of concerns.

The Edge (Gateways)

Public-facing. High concurrency.

  • Handle thousands of simultaneous connections
  • Terminate TLS and validate requests
  • Translate HTTP to internal protocols
  • No database credentials — can't leak what they don't have

Optimised for memory and connection handling.

The Core (Workers)

Private network. Heavy lifting.

  • Execute business logic and database queries
  • Handle computationally expensive operations
  • Manage secrets and credentials securely
  • Not publicly accessible — isolated by design

Optimised for CPU and throughput.

Custom Overlay Network (VL1)
Gateways and Workers communicate via encrypted mesh protocol — not standard HTTP.

Zero Trust Security

Defence in depth. Privilege separation by design.

Credential Isolation

Public-facing gateways don't possess database passwords or signing secrets.
If a gateway is compromised, the attacker cannot directly access backend systems.
The DMZ pattern — applied at the application level.

Protocol Break

The transition from public HTTP to internal mesh protocol acts as a natural firewall.
Malformed payloads and standard web exploits don't propagate to the backend.
Serialisation (HTTP → Protobuf) enforces strict schema validation.

No Lateral Movement

Each request is validated independently. Nodes don't trust each other implicitly.
Even if one component is compromised, it can't freely access other services.
High resistance to data exfiltration from the edge.

The Trade-off (And Why It's Worth It)

The mesh architecture introduces ~2-5ms overhead per request. This "mesh tax" buys:

Scalability — Gateways and Workers scale independently

Security — Privilege separation and protocol isolation

Resilience — Failed components don't cascade

User perception: Negligible. Feels instant for standard web interactions.

Dual-Mode Operation

Distributed by default. Monolith when needed.

Mesh Mode (Standard)

Services distributed across multiple nodes. Gateways communicate with Workers via the overlay network.

  • Horizontal scaling
  • Geographic distribution
  • Independent component updates

Direct Mode (Failover)

"Logical monolith" mode. Transport layer bypassed. Gateways execute backend logic via direct function calls.

  • Zero network dependency
  • Emergency continuity
  • "Break glass" disaster recovery

Same business logic runs in both modes. No code changes required for failover.

How a Request Flows

From user click to database and back.

1
Ingress — User request hits Gateway (HTTPS)
2
Translation — Gateway validates and converts to Protobuf
3
Discovery — Gateway queries directory to find healthy Worker
4
Transport — Secure VL1 session established with Worker
5
Execution — Worker executes logic, queries database
6
Response — Result flows back: Worker → Mesh → Gateway → User
